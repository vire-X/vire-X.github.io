常见的dos命令

2019-11-29
Word count: 517 | Reading time: 1min

JDK、JRE、JVM的关系

JDK=JRE+开发工具集(例如java编译工具等)

JRE=JVM+javaSE标准类库

常见的dos命令

–> dir 列出当前目录下的文件以及文件夹

–> d: 切换到D盘

–>cls 清屏

–>md 创建目录

–>rd 删除目录

–>cd 进入指定目录

–>cd.. 退回到上一级目录

–>cd\ 回到根目录

–>del 删除文件

–>exit 退出dos命令行

注释

1、单行注释 – //

2、多行注释 – /* */

3、文档注释（java特有）

文档注释（java特有）

格式：

/**

*@author 指定java程序的作者

*@version 指定源文件的版本

*@param 方法的参数信息说明

*/

示例：

/**这是我的第一个java程序

@author qjj

@version v1.0

*/

文档注释内容可以被javadoc所解析，生成一套以网页文件形式体现的该程序的说明文档，文档注释是可以被读到的。

注：

1.对于单行和多行注释，被注释的文字不会被JVM解释执行。

2.多行注释里不允许有多行注释嵌套

多注释，多注释，多注释，不然过两天自己都看不懂

输出简单程序hello world

总结：

1.源文件以.java结尾。

2.编译–>javac 源文件名.java

运行–>java 类名

3.源文件中可以有多个class声明的类，编译后，会生成一个或多个字节码文件，字节码文件名与java源文件名中的类名相同。

4.类中可以有主方法（main（）方法），其格式是固定的：public static void main(String[ ] args){}

5.main()方法是程序的入口，方法内是程序的执行部分（有点像c语言的函数）

6.一个源文件中只能有一个声明为public 的类，同时要求此类的类名与源文件名一致。

7.每个语句都以；结束。

8.执行程序：编译：javac.exe 编译完，生成诸多个.class字节码文件。 运行java.exe】

9.System.out.println() : 先输出再换行

System.out.print() : 只输出
